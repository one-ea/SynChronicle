# Responsive Workbench Redesign

## Goal

修复 SynChronicle 创作台在不同视口下的错位、挤压和操作层级混乱问题。桌面端采用稳定编辑工作室，平板端采用画布加抽屉，移动端采用创作优先的单任务模式。

## Scope

- 优化创作台顶栏、章节导航、正文创作区、运行状态区和移动端底部导航。
- 统一响应式断点和面板状态管理。
- 移除桌面端栏宽滑块与布局弹层。
- 保留现有业务 API、运行控制、实时事件、章节选择和模型配置能力。
- 项目列表、登录页、设置页和管理页仅处理此次变更引起的共享样式回归。

## Responsive Modes

### Mobile: 320-767px

- 默认显示创作面板，正文阅读和创作指令输入获得主要空间。
- 底部任务导航包含“章节”“创作”“运行”，每次仅显示一个面板。
- 创作输入区固定在底部导航上方，并适配软键盘和 `env(safe-area-inset-bottom)`。
- 顶栏显示返回作品、当前章节标题和上下文操作入口。
- 运行面板使用移动端卡片流展示连接、进度、Agent、用量和控制操作。
- 运行摘要可在创作面板顶部以紧凑状态显示，例如“运行中 · 87 分”。

### Tablet: 768-1199px

- 中央创作画布始终可见。
- 顶栏提供“章节”和“运行”两个抽屉触发按钮。
- 左右抽屉互斥，打开一个抽屉时关闭另一个。
- 抽屉提供遮罩、显式关闭按钮、外部点击关闭和 `Escape` 关闭。
- 抽屉打开后焦点进入抽屉，关闭后返回原触发按钮。
- 创作输入区固定在画布底部，不受抽屉滚动影响。

### Desktop: 1200px+

- 使用稳定三栏布局：左侧章节栏约 256px，右侧运行栏约 320px，中间创作区弹性增长。
- 两侧栏支持展开和折叠，不支持任意栏宽调整。
- 折叠状态保留图标、关键数字和展开按钮。
- 中央正文内容宽度限制在舒适阅读范围，超宽屏通过增加外侧留白扩展。
- 1600px 以上保持侧栏宽度和正文行宽稳定。

## Component Design

### Workbench Shell

`WorkbenchPage` 继续持有共享业务状态，并向布局壳层提供当前响应模式、活动面板、抽屉和折叠状态。布局模式通过媒体查询与 `matchMedia` 同步，业务数据保持单一来源。

### Project Navigation

- 桌面端作为固定左栏展示。
- 平板端作为左侧抽屉展示。
- 移动端作为独立章节任务页展示。
- 长章节标题允许换行，序号和状态保持可辨识。

### Writing Canvas

- 在所有响应模式中保持主要语义区域和稳定阅读宽度。
- 标题、正文、实时增量和事件流使用一致的排版层级。
- 输入区与正文滚动区分离，正文滚动不会移动主要发送操作。

### Run Status

- 桌面端作为固定右栏展示。
- 平板端作为右侧抽屉展示。
- 移动端改为卡片化运行任务页。
- 创建运行、运行控制、用户问答、模型切换和诊断按操作频率分组。
- 连接状态、当前阶段和评分构成首屏摘要，低频配置位于后续分组。

### Navigation Controls

- 移动端底部导航使用图标、文字和背景共同表达当前状态。
- 平板抽屉触发按钮显示当前章节或运行摘要。
- 桌面侧栏折叠按钮使用明确方向图标和可访问名称。

## Layout Safety

- Grid 和 Flex 的可收缩子项统一使用 `min-width: 0`。
- 长标题、Agent 名称、错误信息和诊断内容允许换行。
- 每个模式都禁止页面级横向滚动。
- 面板内部拥有独立滚动容器，顶栏、输入区和移动导航保持稳定。
- 可交互控件最小触控尺寸为 44px，主要移动操作为 48px。
- 弹层和抽屉不得超出视口，并适配安全区域。

## State Transitions

- 从移动端切换到平板或桌面时，保留当前章节和运行数据，关闭移动任务切换产生的临时面板状态。
- 从桌面切换到平板时，折叠状态不自动打开抽屉。
- 平板抽屉切换始终互斥。
- 浏览器前进和后退继续同步章节与移动活动面板。
- 用户操作产生的焦点在布局切换后移动到当前可见主区域。

## Error Handling

- API 和实时连接错误在当前可见模式内展示，不依赖隐藏面板。
- 移动创作页显示简短错误摘要，并提供跳转运行页的入口。
- 抽屉内发生错误时保持抽屉打开，直到用户处理或主动关闭。
- 提交、运行控制和模型切换继续防止重复操作，并保留 `aria-live` 状态通知。

## Accessibility

- 抽屉使用对话框语义、可访问名称和焦点约束。
- `Escape`、遮罩和关闭按钮均可关闭抽屉。
- 移动底部导航使用 `aria-current` 标记当前任务。
- 折叠侧栏保留可理解的按钮名称和状态。
- 支持键盘导航、缩放到 200% 和 `prefers-reduced-motion`。

## Testing

### Component Tests

- 响应模式状态与面板可见性。
- 桌面侧栏折叠和恢复。
- 平板抽屉互斥、关闭方式和焦点恢复。
- 移动底部导航与浏览器历史同步。
- 创作输入在各模式中的可达性和重复提交保护。

### Browser Tests

- 覆盖 375、768、1024、1200、1440 和 1920px。
- 每个视口检查页面级横向溢出为零。
- 验证桌面三栏尺寸和正文阅读宽度。
- 验证平板抽屉位于视口内且不会改变画布宽度。
- 验证移动导航、固定输入区和安全区域间距。
- 验证长标题、长状态消息和较短视口下的滚动可达性。

### Regression Gates

- Workbench 单元测试和 WCAG AA 检查通过。
- Responsive Playwright 项目全部通过。
- TypeScript 类型检查和生产构建通过。
- 登录、项目列表、设置和管理页面无共享样式回归。

## Acceptance Criteria

- 375-1920px 范围内无页面级横向滚动或面板重叠。
- 移动端进入作品后默认聚焦创作，输入区和底部导航始终可达。
- 平板端中央画布常驻，章节和运行抽屉互斥且具备完整焦点管理。
- 桌面端使用稳定三栏并支持侧栏折叠，正文行宽保持可读。
- 运行创建、运行控制、用户问答、模型切换和诊断在所有模式下可完成。
- 响应式测试、可访问性测试、类型检查和生产构建全部通过。
